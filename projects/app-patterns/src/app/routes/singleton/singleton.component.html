La primera caracteristica de este patron es que el constructor tiene el modificador de acceso "privado" es decir ninguna otra entidad puede instanciar esta clase a traves de sus constructor.

La segunda caracteristica que define al patron singleton es declarar un metodo static que retornara la instancia de si misma, Este sera el metodo con que utilizaran los modulos y componentes que necesiten acceder a la configuracion.

Para obtener una instancia de la clase Config es necesario invocar el unico miembro publico que lo conforma el metodo "getInstance"

Usando el patron singleton garantizas de que solo exista una unica instancia de una clase.
